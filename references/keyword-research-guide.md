# 找词实战指南

## 新词新站核心逻辑

新词 = 过去12个月内出现的关键词（尤其是AI相关）

为什么新词适合新站？
1. 老站还没有针对这些词优化
2. 竞争页面少，intitle结果<1000
3. Google算法对新内容友好
4. 先发优势明显

## 找词渠道详解

### 1. Google Trends

**用法**：
- 输入种子词，查看「相关查询」
- 筛选「上升」标签
- 关注"Breakout"关键词

**实战技巧**：
- 每天刷一遍相关行业的趋势
- 设置邮件提醒
- 结合新闻热点找词

### 2. Semrush Keyword Magic Tool

**筛选设置**：
- KD: 0-30
- Volume: 100+
- Word count: 2-5个词（长尾词）

**导出后处理**：
1. 按主题归类
2. 标记难度等级
3. 标注商业意图

### 3. Similarweb竞品分析

**步骤**：
1. 找到竞品网站
2. 查看「搜索」→「关键词」
3. 筛选低KD词
4. 复制到自己词库

### 4. Reddit/Twitter/HackerNews

**监控方法**：
- 订阅相关subreddit
- 关注行业KOL
- 设置关键词提醒

### 5. Vercel子域名发现

**原理**：
很多新站先用Vercel子域名测试，可以通过搜索找到新上线的网站，反推他们做的关键词。

**搜索语法**：
```
site:vercel.app "你的行业关键词"
```

## 关键词分类标准

| 类型 | KD范围 | 搜索量 | 用途 |
|------|--------|--------|------|
| 核心词 | 20-30 | 1000+ | 首页目标 |
| 中等词 | 10-20 | 500-1000 | 分类页 |
| 长尾词 | 0-10 | 100-500 | 内容页 |
| 微长尾 | 0-5 | <100 | 大量铺量 |

## 词库管理模板

```
| 关键词 | KD | 搜索量 | 难度 | 优先级 | 分配页面 | 状态 |
|--------|-----|--------|------|--------|----------|------|
| xxx | 15 | 320 | 中 | P1 | 首页 | 待建站 |
```

## 判断词是否值得做的 checklist

- [ ] KD < 30
- [ ] 搜索量 > 100/月
- [ ] intitle结果 < 1000
- [ ] 趋势上升或稳定
- [ ] 有商业变现潜力
- [ ] 能在3小时内完成页面
- [ ] 有至少5个相关长尾词可扩展

满足5条以上 → 值得做
