## 附录A：哥飞SEO核心思维体系

> 来源：《从搜索引擎底层逻辑出发 重构你的SEO思维体系》哥飞 2026.03.14

### 一、SEO的三个本质

#### 1. 互联网的本质
> **无数靠链接连起来的网页组成的巨大网络**

| 数据 | 规模 |
|:---|:---|
| 全球网站数量 | **11亿** |
| 全球网页数量 | **1000亿+** |
| 谷歌索引网页 | **500亿+** |

#### 2. 搜索引擎的本质
> **通过找到并收录尽可能多的网页，来满足各种稀奇古怪的搜索需求**

谷歌追求的两大目标：
- ⚡ **最新**：满足时刻出现的全新搜索需求
- 🎯 **最全**：满足每个用户各种稀奇古怪需求

#### 3. SEO的本质
> **让搜索引擎更好的抓取和读懂你的网页**

| 维度 | 具体做法 |
|:---|:---|
| **更好地抓取** | 网站结构清晰、内链完善、sitemap齐全，让爬虫无障碍爬遍每一个页面 |
| **更好地读懂** | 标题、描述、正文关键词准确，让搜索引擎理解你的页面在讲什么 |

### 二、搜索引擎工作流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  爬取   │ → │  解析   │ → │  索引   │ → │  排序   │ → │ 用户点击 │
│ (Crawl) │    │(Parse)  │    │(Index)  │    │ (Rank)  │    │(Click)  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

| 步骤 | 说明 | 关键注意点 |
|:---|:---|:---|
| **爬取** | 爬虫发GET请求，获取HTML代码 | **只是获取，并不执行JavaScript** |
| **解析** | 提取网页主体内容、Meta信息、页面中所有链接 | 标题、描述必须准确 |
| **索引** | 分词处理 → 提取网页主题 → 建立索引 | 正排索引 + 倒排索引 |
| **排序** | 多因素排序 → 呈现搜索结果 | **排名即流量** |
| **用户点击** | 大部分人只看第一页，甚至只点前三个 | 进前10才有流量 |

### 三、关键认知

#### 站长与搜索引擎的关系
> **生态互补关系** —— 迎合搜索引擎，跟搜索引擎**交朋友**，而不是对抗

#### AI来了，SEO已死？
> **我不同意这个说法** —— 说这话的人，明显是**既不懂SEO，也不懂AI**

- AI能满足：部分**资讯信息类**检索需求
- AI无法满足：**社交、电商、游戏、工具、服务、影视**等重交互需求

### 四、通过SEO搞到流量的三件事

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  使劲挖需求   │ → │  不断上页面   │ → │  持续加外链   │
│  关键词的背后  │    │ 一个关键词主题  │    │ 虽说不是有了   │
│  是需求，同一  │    │ 一个页面，理解  │    │ 外链就一定行，  │
│  个需求会有不同 │    │ 需求后为每一个  │    │ 但是没有外链大  │
│  的关键词说法  │    │ 有搜索量的关键  │    │ 多数时候都拿不  │
│             │    │ 词做出SEO友好  │    │ 到排名        │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 五、12字秘诀：需求关键词挖掘

> **词找词、词找站、站找词、站找站**

| 方法 | 说明 | 工具/技巧 |
|:---|:---|:---|
| **词找词** | 用一个词找到更多相关词 | 谷歌下拉搜索、相关搜索、关键词规划工具 |
| **词找站** | 用一个词找到排名好的网站 | 搜索关键词，分析前10名站点 |
| **站找词** | 用一个网站找到它排名的所有词 | Ahrefs/Semrush Organic Keywords |
| **站找站** | 用一个网站找到更多同类竞争对手 | Similarweb 竞品分析 |

### 六、SEO是动态竞争

> **每一个关键词的搜索结果都是一个排行榜，你和排行榜上的其他页面在PK**

| 经验 | 说明 |
|:---|:---|
| **链接传递权重** | 任何链接都能够传递权重，不管是**内链还是外链** |
| **页面为最小单位** | 谷歌的最小排名单位是**页面**，首页也是一个特殊的页面 |
| **竞争力是前提** | 任何页面能拿到排名的**前提条件**都是**竞争力足够** |
| **搜索结果是排行榜** | 任何关键词搜索结果都是一个**排行榜**，页面之间在**相互竞争** |

## 附录B：域名切换SEO指南

> 来源：《如何降低域名切换对SEO的效果影响》2026中国谷歌SEO大会

### 一、为什么域名切换会影响SEO

| 因素 | 说明 |
|:---|:---|
| **URL变化** | Google需要重新识别页面身份 |
| **权重传递** | 旧域名积累的权重与信任需要时间迁移到新域名 |
| **页面变化** | 重定向、抓取、索引替换都需要时间完成 |

> **核心结论**：影响SEO的本质不是"权重突然消失"，而是"搜索引擎需要时间重新确认新旧页面的继承关系"

### 二、域名迁移的常见风险

| 风险类型 | 具体表现 |
|:---|:---|
| **重定向错误** | 旧页面跳错页、跳404、全部跳首页 |
| **索引请求遭拒** | 新站遗留noindex或robots屏蔽 |
| **信号说明错误** | canonical、sitemap、内链未同步更新 |
| **大幅修改** | 同时修改模板、URL结构、导航等 |

### 三、降低SEO影响的四大核心原则

1. **旧URL与新URL的继承关系要清晰**
2. **用301永久重定向传递明确迁移信号**
3. **canonical、内链、sitemap等站内信号必须统一**
4. **迁移是持续监控与修正的过程，不是一次性动作**

### 四、迁移前准备

| 步骤 | 具体内容 |
|:---|:---|
| 1 | **URL映射表** — 建立旧URL到新URL的一一对应关系 |
| 2 | **内容备份** — 备份旧站所有内容和数据库 |
| 3 | **新站搭建** — 新站完全复制旧站内容和结构 |
| 4 | **技术检查** — 新站无noindex、robots允许抓取 |
| 5 | **重定向规则** — 准备好301重定向规则 |
| 6 | **监控工具** — 确保GSC、GA等工具可以正常监控 |

### 五、迁移中操作

| 顺序 | 操作 |
|:---|:---|
| 1 | **启用301重定向** — 旧站所有页面启用301跳转到对应新页面 |
| 2 | **更新DNS** — 如有需要，更新DNS指向 |
| 3 | **提交新sitemap** — 在GSC提交新域名的sitemap |
| 4 | **验证抓取** — 用GSC的URL检查工具验证新页面可被抓取 |
| 5 | **检查重定向** — 用curl或工具检查301是否正确生效 |

### 六、迁移后监控

| 时间段 | 关注重点 |
|:---|:---|
| **第1周** | 新站是否被正常抓取和索引 |
| **第1个月** | 排名和流量变化趋势 |
| **第3个月** | 流量是否恢复到迁移前水平 |
| **第6个月** | 长期效果评估 |

# 建站三字经 SOP（正文）

## 阶段一：建站之初（筛词库 + 怼布局）

### 1.1 筛词库

**目标**：找到适合新站的低竞争关键词

**工具**：
- Google Trends - 趋势验证（三个本质：判断需求趋势）
- Semrush Keyword Magic Tool
- Ahrefs Keywords Explorer

**筛选标准**：
| 指标 | 要求 | 说明 |
|------|------|------|
| KD (关键词难度) | < 30 | 新站有机会排名 |
| 搜索量 | > 100/月 | 有基本流量价值 |
| intitle结果 | < 1000 | 竞争页面少 |
| 趋势 | 近12月上升 | 新词新站机会 |
| 商业意图 | 中高 | 便于后期变现 |

**操作步骤**：
1. 在Semrush输入种子词，导出相关关键词列表
2. 按KD升序排列，筛选KD<30的词
3. 用Google Trends验证趋势（近期开始上升为佳）
4. Google搜索 `intitle:"关键词"` 查看标题竞争度
5. 将筛选出的词按主题归类，建立词库表格

**12字秘诀应用**：
- **词找词**：用种子词找到更多相关词
- **词找站**：搜索关键词，分析前10名站点
- **站找词**：用Ahrefs查看竞品排名的所有词
- **站找站**：用Similarweb找到更多竞争对手

### 1.2 怼布局

**目标**：将词库合理分配到网站各页面

**页面类型分配**：
- **首页**：核心大词（1-3个）
- **分类页**：中等竞争度词
- **内容页/工具页**：长尾词（每个页面聚焦1个主词）

**TDH优化**（让搜索引擎更好地读懂）：
- **Title**：主关键词 + 修饰词（60字符内）
- **Description**：包含关键词的行动号召（160字符内）
- **Headings**：H1包含主词，H2-H3覆盖长尾变体

**内链结构**（让爬虫更好地抓取）：
```
首页
├── 分类A
│   ├── 页面A1
│   └── 页面A2
├── 分类B
│   ├── 页面B1
│   └── 页面B2
└── ...
```

## 阶段二：看收录（收录稳，上初链）

### 2.1 提交收录

**上线后立即执行**：
1. 提交Google Search Console
2. 提交sitemap.xml
3. 在首页放置Google Analytics代码

### 2.2 监控收录

**观察指标**：
- GSC中「网页」→「已编入索引」数量
- `site:你的域名` 搜索结果数

**收录正常信号**：
- 首页24-48小时内被收录
- 内页7-14天内陆续收录
- 收录数量稳定增长

### 2.3 上初链（初步外链）

**触发条件**：收录稳定增长2周以上

**初链渠道**（免费/低成本）：
| 渠道 | 方法 | 数量 |
|------|------|------|
| 社交媒体 | Twitter/X、Reddit相关板块分享 | 5-10条 |
| 书签网站 | Delicious、Pocket等 | 3-5条 |
| 目录提交 | DMOZ替代品、行业目录 | 5-10条 |
| 论坛签名 | 相关行业论坛 | 3-5条 |
| 博客评论 | 行业博客有价值评论 | 10-20条 |

**注意**：初链不求质量，但求自然、相关、稳定

---

## 阶段三：看反馈（反馈佳，加页面）

### 3.1 观察正反馈

**正反馈信号**：
- GSC展示次数上升
- 点击数开始增长
- 部分词进入前100名
- 用户停留时间>30秒

### 3.2 加页面（扩大词库覆盖）

**策略**：
1. 在原词库基础上，挖掘更多长尾变体（词找词）
2. 每个新页面聚焦1个长尾词
3. 保持内容质量，避免垃圾页面

**内容生产节奏**：
- 初期：每周2-5页
- 稳定期：每周5-10页
- 爆发期：根据资源决定

---

## 阶段四：看词库（词库涨，上优链）

### 4.1 监控词库增长

**工具**：
- Semrush → Organic Research → Positions
- Ahrefs → Organic Keywords

**健康指标**：
- 每周新增排名关键词 > 10个
- 排名前20的词稳步增加
- 总排名词库持续增长

### 4.2 上优链（优质外链）

**触发条件**：词库连续增长1个月以上

**优质外链渠道**：
| 渠道 | DA要求 | 获取方式 |
|------|--------|----------|
| 客座博客 | DA>30 | 主动联系站长 |
| 资源页链接 | DA>40 | 提供有价值资源 |
| 行业目录 | DA>35 | 高质量付费目录 |
| 新闻稿发布 | DA>50 | PRWeb等付费渠道 |
| 断链建设 | DA>30 | 用Ahrefs找断链 |

**首页外链策略**：
- 每月2-5条高质量外链到首页
- 锚文本多样化（品牌词、裸链、部分匹配）
- 避免过度优化（精确匹配锚文本<20%）

---

## 阶段五：看排名（排名升，调内容）

### 5.1 关注前20名词库

**重点监控**：
- GSC中排名11-20的词（最容易进首页）
- 排名波动大的词
- 有搜索量但排名靠后的词

### 5.2 On-Page SEO优化

**分析首页前5名**：
1. 查看他们的Title/Description写法
2. 分析内容结构和长度
3. 检查内链布局
4. 观察页面加载速度
5. 研究用户体验设计

**优化动作清单**：
- [ ] 内容长度达到或超过前5名平均
- [ ] 增加相关LSI关键词
- [ ] 优化图片alt标签
- [ ] 添加FAQ部分（获取富媒体摘要）
- [ ] 提升页面加载速度（<3秒）
- [ ] 优化移动端体验
- [ ] 添加内部链接（从已有高权重页面）

## 阶段六：卡细节（观其变，静等候）

### 6.1 调整后的观察期

**关键原则**：调整后不要频繁改动

**观察周期**：
- 小调整：1-2周
- 大改版：3-4周
- 算法更新后：4-6周

### 6.2 排名波动正常心态

**正常波动**：
- 单日排名±5位：正常
- 单周排名±10位：正常
- 算法更新后波动：正常

**何时担心**：
- 连续2周持续下降
- 排名消失（被惩罚信号）
- 流量断崖式下跌

## 常见错误避坑

1. **急于求成** - 新站不要一上来就猛发外链，容易被惩罚
2. **忽视收录** - 没收录就谈排名是空中楼阁
3. **关键词过宽** - 新站不要碰KD>50的词
4. **内容单薄** - 每页至少500字，工具站要有实际功能
5. **外链单一** - 多样化外链来源，避免被算法打击
6. **频繁改版** - 调整后要给Google时间反应
7. **忽视移动端** - 60%+流量来自移动设备

## 进阶技巧

### 快速收录技巧
- 在高权重平台（Reddit、Medium）发布带链接的内容
- 使用Google Indexing API（需申请）
- 提交到新闻网站获取快速索引

### 外链建设心法
- 先给价值，再要链接
- 内容为王，链接自然来
- 质量 > 数量（长期）
- 相关性 > DA值

### 内容生产加速
- 用AI生成初稿，人工润色
- 建立内容模板库
- 外包低价值内容创作
- 批量生产同类页面
